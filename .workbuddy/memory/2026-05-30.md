# 2026-05-30 开发日志

## 21:10 — 调试：完成 G1 后结果页报错

**现象**: 用户完成 G1 后进入 results.html，显示 "页面加载出错" 兜底 UI，但徽章显示 "已完成 1/4"。

**根因推测**:
- `integrator.js` 从 store 读取 G1 数据时，若 `dimensions` 对象存在但缺少预期键（如旧测试数据），`raw[dim]` 保持 `null`
- `translator.js` 的 G1 区块在 `g1Dims` 为空数组时，`reduce` 返回 `undefined`，随后访问 `primary.name` 抛出 `TypeError`

**修复内容**:
1. `results.html`: catch 块现在显示具体错误信息（`err.message`），方便后续排查
2. `js/layers/translator.js`: 为 G1/G2/G3/G4 各区块添加 `gXDims.length === 0` 的防御分支，避免空数组导致 `reduce` 崩溃
3. `js/layers/integrator.js`: 增加 `data.gameX.dimensions` 存在性检查、`Number.isNaN` 过滤、`hasAny` 警告日志
4. `js/ui/portrait.js`: `formatSummary` 增加空值保护

**产出文件**:
- `results.html`
- `js/layers/translator.js`
- `js/layers/integrator.js`
- `js/ui/portrait.js`
