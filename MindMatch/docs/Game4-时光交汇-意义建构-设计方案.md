# Game 4「时光交汇·六帧手记」设计方案

> 定位：G1（现在/核心驱动）→ G2（过去/职业锚）→ G3（未来/认知风格）→ **G4（整合/意义建构）**
> 心理理论：Steger MLQ 生命意义双维模型（Presence + Search）
> 交互方式：双重反应（感受反应 + 追问反应）
> 叙事时间线：时间交汇（过去+现在+未来的场景汇聚）
> 状态：方案设计完成，待实施

---

## 一、设计定位

| 属性 | 值 |
|------|-----|
| **学术框架** | Steger MLQ (2006) — 拥有意义感 (Presence) + 寻求意义倾向 (Search)，正交双维 |
| **游戏类型** | 场景叙事 + 双重直觉反应（非 SJT 选择，非排序，非抛弃） |
| **输出** | 2 个维度 × 0-10 连续分数 + 行为摘要 |
| **目标时长** | 3-5 分钟（6 个场景 × ~40 秒/个） |
| **核心叙事** | 时间在这里汇聚。过去、现在、未来的记忆交织在一起，你只需要感受——每个画面带给你的第一反应 |
| **交互策略** | 新建 `dual-reaction.js` 策略 |

---

## 二、叙事设定

> 时光在这里交汇。
> 你走过的路（过去）、正在走的路（现在）、可能走的路（未来），此刻交织在一起。
> 我们从记忆的河流里捞出了 6 个画面——它们是真实的、具体的。
> 不需要分析，不需要思考对错。
> 看完每个画面，感受你的第一反应。

**隐喻解析**：
- 时光交汇 = 整合前三场游戏的时间线（G1 现在 → G2 过去 → G3 未来 → G4 汇聚）
- 六帧手记 = 像翻照片册一样，每个场景有画面、有人物、有情绪
- 双重反应 = 看完一段回忆后自然产生的两个心理反应（不是做题，是感受）

**与 G1/G2/G3 的差异化**：

| | G1 | G2 | G3 | **G4** |
|---|---|---|---|---|
| 机制 | 情境选择 | 多阶段抛弃 | 归类+排序 | **双重反应** |
| 体验感 | "做决策" | "舍弃" | "认识自己" | **感受+追问** |
| 数据类型 | 选项权重 | 放弃=扣分 | 分组+排名 | **独立双维度** |
| 判断难度 | 中 | 中 | 中 | **低（直觉反应）** |

## 三、交互设计：双重反应机制

### 为什么用双重反应？

传统 MLQ 量表用 Likert 5/7 点测两个维度（"我理解生命的意义" → Presence），但抽象陈述句放到游戏里会破坏沉浸感。

本设计改为：**每个具体场景后，问两个独立的直觉问题**，分别捕获 Presence 和 Search。用户不会觉得自己在"做题"，更像是看完一段回忆后自然产生的心理反应。

### 交互流程

```
叙事引导页 → 场景1(读故事+反应①+反应②) → 场景2 → ... → 场景6 → 完成页
```

### 每个场景的标准交互

```
┌─────────────────────────────────────────────┐
│  G4 时光交汇                ● ○ ○ ○ ○ ○   1/6 │
│─────────────────────────────────────────────│
│                                              │
│  [场景标题]                                  │
│                                              │
│  [场景故事文案，3-4行，有画面感]              │
│                                              │
│  ─── 看完这段记忆，你的心里更接近 ───         │
│  ┌──────────────────┐  ┌──────────────────┐  │
│  │ "虽然辛苦，       │  │ "就……普通的一天  │  │
│  │  但没白过"       │  │  而已"           │  │
│  └──────────────────┘  └──────────────────┘  │
│                                              │
│  ─── 看完之后，你更想 ───                    │
│  ┌──────────────────┐  ┌──────────────────┐  │
│  │ "想想接下来往哪走"│  │ "别想太多，       │  │
│  │                   │  │  回家吃饭"        │  │
│  └──────────────────┘  └──────────────────┘  │
│                                              │
└─────────────────────────────────────────────┘
```

### 反应规则

| 步骤 | 问什么 | 衡量维度 | 操作 |
|------|--------|---------|------|
| 反应 ① | "看完这段记忆，你的心里更接近…" | **Presence**（拥有意义感） | 二选一，点击后高亮 |
| 反应 ② | "看完之后，你更想…" | **Search**（寻求意义倾向） | 二选一，点击后高亮 |

- 反应 ① 和反应 ② **先后呈现**（读完场景 → 先问感受 → 再问追问）
- 两个反应**独立计分**，不互斥
- **无正确答案**，无反馈文案（保持沉浸感，不在 G4 做认知干扰）
- 选项顺序每次随机打乱（避免位置效应）

## 四、6 个具体场景

每个场景来自不同时间线，呼应 G1-G3 的叙事。每个场景配两个独立反应。

### 场景 1「转正那天傍晚」— 现在（呼应 G1）

> 你收到转正邮件了。同事们陆续走了，办公室安静下来。你关掉电脑，看见桌上还放着入职那天自己写的便签——上面写着"加油"。你把它放进了抽屉，关灯下楼。出了大楼，抬头看了一眼晚霞。

| 反应 | 选项 A（+） | 选项 B（-） | 维度 | 选项顺序随机 |
|------|-----------|-----------|------|:---:|
| ① 感受 | 🌕 "虽然辛苦，但这段日子没有白过" | 🌑 "就……普通的一天而已" | Presence | ✅ |
| ② 追问 | 🔭 "回家路上想想，接下来要往哪走" | 🚶 "回家吃饭，别想太多" | Search | ✅ |

### 场景 2「毕业离校的那个早晨」— 过去（呼应 G2）

> 你拎着行李箱走出宿舍楼。室友在群里发了条语音说"保持联系"。你走到校门口，回头望了一眼——食堂、图书馆、操场上晨跑的人。你转身走向公交站。车来了，你刷了卡，找了个靠窗的位子坐下。

| 反应 | 选项 A（+） | 选项 B（-） | 维度 | 选项顺序随机 |
|------|-----------|-----------|------|:---:|
| ① 感受 | 🌕 "那些日子很珍贵，我把它们记在心里了" | 🌑 "终于走了，没什么好留恋的" | Presence | ✅ |
| ② 追问 | 🧭 "毕业后我到底想过什么样的生活？" | 🍃 "以前的日子值得回味，但未来才是重点" | Search | ✅ |

### 场景 3「项目上线后的凌晨」— 现在（呼应 G1）

> 你和团队通宵到凌晨四点，终于上线了。有人在群里发了庆祝红包。你抢了一个两毛的，然后趴在桌上。旁边同事拍你肩膀说"辛苦了"。你没抬头，但嘴角动了一下。

| 反应 | 选项 A（+） | 选项 B（-） | 维度 | 选项顺序随机 |
|------|-----------|-----------|------|:---:|
| ① 感受 | 🌕 "累，但值了。这件事做成的感觉很好" | 🌑 "做完了，但说实话没什么成就感" | Presence | ✅ |
| ② 追问 | 💭 "忙完这一阵，我真正想做的是什么？" | 🎯 "好好休息，明天继续冲下一个项目" | Search | ✅ |

### 场景 4「一次偶然的行业分享」— 触发探索

> 周末你随手点开了一个线上分享直播。台上一个前辈讲了自己从"每天机械重复"到"找到真正热爱的事"的故事。他说："不是所有人都能一上来就找到意义，但至少你要在找。"你听了四十分钟，关掉直播，在窗边站了一会儿。

| 反应 | 选项 A（+） | 选项 B（-） | 维度 | 选项顺序随机 |
|------|-----------|-----------|------|:---:|
| ① 感受 | 🌕 "他的故事让我想起了自己的一些经历" | 🌑 "别人的故事是别人的，跟我关系不大" | Presence | ✅ |
| ② 追问 | 🔭 "这句话点了我一下，我今晚得好好想想" | 📱 "看看下一个视频推送什么" | Search | ✅ |

### 场景 5「被分配到一个新项目组」— 现在（呼应 G1）

> HR 发了邮件通知你加入新项目。你打开邮件：新领域、新团队、新挑战。你往下滚动看了看项目周期——比现在的项目长一倍。你关掉邮件，喝了一口水，打开了新项目的文档。

|------|-----------|-----------|------|:---:|

| ② 追问 | 🔍 "这个新领域能让我发现什么不一样的吗？" | 📋 "先看看文档里要我做什么，做就是了" | Search | ✅ |

### 场景 6「离职同事的告别饭局」— 人际连接

> 合作了两年的同事要转岗了。饭桌上大家聊了很多以前的事。他说："你以后肯定能做得更好。"回去的路上，你一个人走在街道上，风有点凉。你想起了刚入职时他带你熟悉系统的那天下午。

| 反应 | 选项 A（+） | 选项 B（-） | 维度 | 选项顺序随机 |
|------|-----------|-----------|------|:---:|
| ① 感受 | 🌕 "有人记得你、认可你，这很重要" | 🌑 "人来了走了，很正常" | Presence | ✅ |
| ② 追问 | 🧭 "他转了，我呢？我该往哪走？" | 🌅 "这些关系是支撑我的东西" | Search | ✅ |

## 五、评分计算

### 维度得分

```
6 个场景 × 2 个反应 = 12 个数据点
每个数据点: 选项 A → +1，选项 B → -1

Presence = Σ(反应①) → 范围 [-6, +6]
Search   = Σ(反应②) → 范围 [-6, +6]

normalize(x) = (x + 6) / 12 * 10
  -6 → 0（最低）
   0 → 5（中间）
  +6 → 10（最高）
```

### 每个场景的权重设计

所有场景权重相同（每场景 ±1），理由：
- MLQ 的 10 个题项也是等权计分（Steger et al., 2006）
- 不同场景覆盖不同意义来源方向，等权保证方向平衡
- 6 个场景的覆盖度已足够（远超 Cronbach α > 0.70 所需的最小题项数）

### meaningSource 计算（补充数据结构定义中的可选字段）

```javascript
// 根据 Search 得分的场景来源分布判断意义来源方向
// 场景 1,3,5 → 工作/成就方向
// 场景 2,6 → 人际连接方向
// 场景 4 → 探索/自我方向

function calcMeaningSource(decisions) {
  const workSearch = [0,2,4].filter(i => decisions[i].searchReaction === 'A').length;
  const peopleSearch = [1,5].filter(i => decisions[i].searchReaction === 'A').length;
  const selfSearch = [3].filter(i => decisions[i].searchReaction === 'A').length;

  if (workSearch >= 2 && workSearch > peopleSearch && workSearch > selfSearch) return "创造/成就";
  if (peopleSearch >= 1 && peopleSearch > workSearch) return "影响他人";
  if (selfSearch === 1) return "自我探索";
  return "稳定实践";
}
```

映射到 `数据结构定义.md` 的四种来源：
| meaningSource 输出值 | 含义 |
|---|---|
| "影响他人" | 通过人际连接获得意义（场景 2、6 倾向 Search） |
| "创造/成就" | 通过工作和成就获得意义（场景 1、3、5 倾向 Search） |
| "揭示真相" | 通过深度思考获得意义（场景 4 倾向 Search） |
| "稳定实践" | 不太主动寻求，在现有框架中找到满足 |

### profileType 计算

```javascript
// 根据 Presence × Search 交叉分类
function calcProfileType(presence, search) {
  const p = presence >= 5;  // 高 Presence
  const s = search >= 5;   // 高 Search
  if (p && s) return "成长型";    // 感到有意义，仍在探索
  if (p && !s) return "安定型";   // 感到有意义，不再探索
  if (!p && s) return "探索型";   // 没感到意义，正在寻找
  return "游离型";                 // 都低
}
```

### 四种意义取向类型

| 类型 | 条件 | 文案 |
|------|------|------|
| **成长型** | Presence 高 + Search 高 | "你感到当下的方向是有意义的，同时也没有停止寻找更多可能。这种状态很珍贵——既扎根又伸展。" |

## 十、UX 设计要点

### 10.1 视觉风格

- **整体色调**：暖色渐变（从过去暖黄 → 现在中性白 → 未来冷蓝），表现时间交汇
- **场景卡片**：简约叙事风格，类似"手记"——偏日记质感，文字为主，无多余装饰
- **反应按钮**：柔和的卡片式按钮，选中后有温暖的高亮色（非刺眼的绿色/红色）
- **场景过渡**：淡入淡出 + 轻微的横向滑动（像翻相册）

### 10.2 动画设计

| 动作 | 动画 | 时长 |
|------|------|------|
| 场景切入 | 淡入 + 轻微上移 | 600ms |
| 反应①选项出现 | 从下方滑入 | 400ms |
| 反应②选项出现 | 反应①选中后出现 | 300ms |
| 选项选中 | 高亮 + 轻微放大 | 200ms |
| 场景切出 | 淡出 | 400ms |
| 完成页 | 中央渐显 + 文字逐行出现 | 800ms |

### 10.3 关键 UX 决策

| 决策 | 选择 | 理由 |
