# 数据结构定义 —— 游戏层输出

> 版本: v1.1 | 日期: 2026-05-30 | 用途: Day 1 任务 1.6 产出物
> 原则: 每个游戏输出多维连续分数剖面（非单一标签），为 AI 肖像层提供深度数据

---

## 〇、总体设计合理性说明

### 为什么选"连续分数剖面"而非"单一分类标签"？

心理测量学共识：人格和动机特质在人群中呈**连续分布**，而非离散类别。将连续变量强行二分为"成就型/非成就型"会丢失 67% 以上的个体差异信息（MacCallum et al., 2002, *Psychological Methods*）。具体到本项目：

| 设计选择 | 合理性依据 |
|---------|-----------|
| 连续剖面（14维，0-10 分） | 每个人的动机是混合的——一个人可以既高成就又高亲和。单一标签掩盖了这种混合剖面的丰富性，而 AI 肖像层需要这种丰富性来生成"被说中"的洞察。 |
| 总分不恒定（三项动机独立计分） | McClelland 本人的理论即强调三动机**非互斥**——与 Maslow 层级不同的是，高成就者完全可以同时拥有高亲和动力。强制归一化（三项加总=10）会制造虚假的权衡。 |
| 行为数据与维度分数分离 | 反应时、悬停日志等行为数据用于**信度验证**（如一致性检验），不直接参与维度计分。这避免了"鼠标抖了一下就改变人格分数"的测量噪声。 |

### 为什么选这四个游戏/框架？

四个游戏分别测量了岗位匹配中**因果链最上游**的四个构念：

```
动机（G1: 你为什么做事）
  → 锚点（G2: 你不放弃什么）
    → 认知（G3: 你怎么处理信息）
      → 意义（G4: 你在找什么）
```

这四个层次与 Barrick & Mount (1991) 元分析中"人格→工作绩效"的因果链一致：动机驱动行为选择，锚点约束职业路径，认知风格决定工作方式，意义取向预测长期留存。

## 一、全局通用行为数据（4 个游戏均采集）

| 字段 | 类型 | 说明 |
|------|------|------|
| `userId` | string | 匿名标识（localStorage 生成） |
| `gameId` | `"game1"` \| `"game2"` \| `"game3"` \| `"game4"` | 游戏标识 |
| `startedAt` | timestamp | 游戏开始时间 |
| `completedAt` | timestamp | 游戏完成时间 |
| `totalDuration` | number (ms) | 游戏总耗时 |
| `decisions` | array | 决策点数组，每个决策点包含： |

**每个决策点数据结构：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `decisionId` | string | 决策点标识（如 "q1"、"q2"） |
| `chosenOption` | string | 用户选择的选项 ID |
| `decisionTime` | number (ms) | 该决策耗时 |
| `hoverLog` | array | `[{optionId, duration}]` 各选项鼠标悬停时长 |

### 合理性说明

**为什么采集"决策耗时"和"悬停日志"？** 这来自 SJT 评分研究的两个共识：

1. **决策耗时作为一致性验证**（Georgiou et al., 2020）：如果一个人声称"帮助他人是我最重要的动力"但在 SV 选项上反应极短（<1s），说明该选择可能属于**社会赞许性反应**（假装自己在乎）。过长耗时（>30s）则暗示犹豫或认知负荷——两者都不影响维度分数本身，但降低该维度的 `consistencyScore`。

2. **悬停日志作为选项探索信号**：如果用户在三个选项上都有显著悬停（而非只看一个就秒选），说明其在**真实权衡**——这反而提高了该决策点的测量信度。这个数据在 Demo 阶段暂不参与计算，但为后期算法优化（如加权每个决策点的信度贡献）预留空间。

**为什么时间戳 + userId 是全局必需的？** 四个游戏独立运行但最终需要合并分析。没有 `userId` 和 `startedAt` 就无法关联同一用户的多游戏数据（数据一体性要求）。`totalDuration` 用于过滤异常快速（<30s 全部点完 = 乱选）或异常慢速（>20min = 中途离开）的记录。

## 三、每个游戏详细数据结构

### G1: 核心驱动力

```
game1_output = {
  gameId: "game1",
  dimensions: {
    nAch: number,    // 成就动机: 追求卓越、克服挑战 (0-10)
    nPow: number,    // 权力动机: 影响控制、追求地位 (0-10)
    nAff: number,    // 亲和动机: 建立关系、被接纳 (0-10)
  },
  behaviorSummary: {
    primaryDriver: string,         // "nAch" | "nPow" | "nAff"
    profileType: string,           // 如 "高成就/中亲和/低权力"
    consistencyScore: number,      // 决策一致性（0-1，越高越一致）
  }
}
```

**评分逻辑**：每个决策点的选项对各维度有不同的权重贡献，累加后归一化至 0-10。三项分数不恒定总和。

**理论来源**：McClelland (1961, 1975, 1987); Managers' Motivation Profiles (Sage, 2018)

### G1 合理性说明

**为什么选 McClelland 三动机而非其他动机理论？**

| 候选框架 | 为什么不选 |
|---------|-----------|
| Maslow 需求层次 | 层级模型（一层满足才到下一层），不适用于**同时测多个动机强度**；且在现代组织心理学中已被更实证的理论取代 |
| Deci & Ryan SDT | 内在/外在动机二分法（加上三种基本心理需求），维度太少（仅 3 个需求），无法区分"追求卓越"和"追求权力"——两者在 SDT 中都算内在动机 |
| McClelland 三动机 | **三个独立正交维度**，已经被大量组织心理学研究验证（Spangler, 1992 元分析; Sage 2018 管理者动机剖面研究），且与岗位匹配有直接映射（Barrick & Mount 元分析中，成就动机→尽责性→任务绩效，亲和动机→宜人性→团队绩效） |

**为什么三项分数不恒定总和？** 这是 McClelland 理论的核心预设——三项动机**非互斥**。一个管理者可以同时高成就（想做好事）、高权力（想带人）、高亲和（想被人喜欢）。强制归一化（如三项加总=10）会制造"零和博弈"假象，而这在现实中不成立。Sage 2018 年的管理者剖面研究证实：高效管理者往往是**高成就+高权力+中等亲和**的混合剖面，而非某一项占绝对主导。

**为什么 G3 可以只保留一个维度但 G1 必须三个都保留？** G3 的"整体-分析"本身就是一个连续统一体（一端到另一端），一个维度就能捕获完整变异。但 McClelland 的三动机之间存在**非线性交互**（如高成就+低亲和与高成就+高亲和的行为模式完全不同），删掉任何一个维度都会破坏肖像的完整性。

**为什么 `consistencyScore` 是必要的？** McClelland 的传统测量方法（TAT 图片投射）本身存在评分者信度问题（α 约 0.65-0.75）。SJT 的表现优于 TAT，但仍然受制于社会赞许性和情境效应。`consistencyScore` 监控用户在同类决策点上是否一致——如果三次成就相关决策选了三个截然不同的方向，该维度的信度存疑。在 Demo 阶段不影响分数，但在 AI 洞察层会触发"决策不一致"提示。

### G2: 不可放弃的锚点

```
game2_output = {
  gameId: "game2",
  dimensions: {
    TF: number,    // 技术/职能型 — 追求专业深度
    GM: number,    // 管理型 — 追求领导统筹
    AU: number,    // 自主/独立型 — 追求工作自由度
    SE: number,    // 安全/稳定型 — 追求可预期保障
    EC: number,    // 创业型 — 追求创造新事物
    SV: number,    // 服务型 — 追求帮助他人
    CH: number,    // 挑战型 — 追求解决难题
    LS: number,    // 生活方式型 — 追求工作生活平衡
  },
  behaviorSummary: {
    primaryAnchor: string,        // 最高分维度
    secondaryAnchor: string,      // 次高分维度
    bottomAnchor: string,         // 最低分维度（最不可能放弃的）
  }
}
```

**评分逻辑**：每个决策点当用户"放弃"某类选项时，对应维度扣分；"坚持"则加分。最终各维度独立 0-10 分。

**理论来源**：Schein (1978, 1990); COI 40 题量表；Worcester 大学实证验证 (2013)

### G2 合理性说明

**为什么选"放弃才扣分"而非"选择才加分"？** 这是 Schein 职业锚理论最核心的洞见：锚点是"你在面临真正取舍时**无论如何都不肯放弃**的东西"（Schein, 1978, p.125）。问卷中选"我喜欢自由"只需要低认知成本，但在游戏中面对"高自由度 vs 高薪水 vs 稳定保障"的二选一——谁被牺牲，谁才是锚点。这个设计将传统的 Likert 量表（"自由对你多重要？1-7"）转化为**行为层面上的偏好揭示**（revealed preference），比自陈报告更抗社会赞许性偏差。

**为什么 Demo 保留全部 8 维而非缩减？** 最初的讨论考虑过合并 SE+LS（安全/生活方式）和 CH+EC（挑战/创业），但 Wheeler et al. (2007) 的因子分析证实 8 个锚点的因子结构是稳定的，合并会丢失区分效度。Demo 阶段的全量采集也为后期数据分析保留了最大灵活性——如果某两个维度高度相关（r > 0.8），可以用主成分分析（PCA）合并，但这需要真实用户数据才能判断。

**为什么 `bottomAnchor` 值得记录？** Schein 在 1990 年的修订中指出，一个人的**最低锚点**（最不可能放弃的东西的对立面——即最无所谓的东西）同样具有诊断价值。例如一个人安全/稳定最低但挑战/创业最高，说明其高度适应不确定性——这是创业型人才的强烈信号。在肖像报告中，"你最不在乎的是什么"往往比"你最在乎什么"更有反差感和洞察力。

**为什么 8 个维度各独立 0-10 而非排名？** 与 G1 的逻辑一致——一个人可以同时坚定地追求多项价值。COI 量表的计分方式也是各维度独立 Likert 得分（Worcester 大学, 2013），而非强迫排序。强迫排序（如 Q-sort）会增加测量信度但丢失"绝对强度"信息——我们不知道一个人的"技术型 8 分"是"非常在乎技术"还是"在所有选项中相对靠前"。

### G3: 认知处理风格

```
game3_output = {
  gameId: "game3",
  dimensions: {
    wholistAnalytic: number,    // 整体-分析连续体 (-1.0 整体 ↔ +1.0 分析)
    fieldIndependence: number,  // [可选] 场独立-场依存 (-1.0 场依存 ↔ +1.0 场独立)
  },
  behaviorSummary: {
    dominantStyle: string,      // "整体型" | "分析型" | "平衡型"
    flexibility: number,        // 认知灵活性 (0-10，越高越能切换风格)
    impulseScore: number,       // 冲动-反思维度 (0-10，越高越冲动)
  }
}
```

**评分逻辑**：通过信息处理任务的反应时比率 + 策略选择计算连续分数。整体型倾向快速把握全局结构，分析型倾向逐步拆解细节。

**理论来源**：Riding CSA; Witkin 认知风格理论；Kagan 反思/冲动型；刘惠军 & 李洋 EEG 验证 (2006)

### G3 合理性说明

**为什么只测"整体-分析"一维（核心）+ "场独立"可选？** Riding & Cheema (1991) 的综述综合了 30+ 种认知风格模型，归结为两个正交维度：

| 维度 | 测量内容 | 岗位匹配价值 |
|------|---------|:---:|
| **整体-分析 (W-A)** | 信息加工策略：全局结构优先 vs 局部细节优先 | **高** — 直接影响工作中的问题解决方式 |
| 言语-表象 (V-I) | 信息表征偏好：文字加工 vs 图像加工 | 低 — 主要影响学习风格，与岗位绩效的相关性弱 |

V-I 维度被排除的原因：Barrick & Mount 元分析中，认知风格维度中仅**信息加工策略**（对应 W-A）与工作绩效有稳定的中等效应量（ρ≈0.15-0.20），而表征偏好（V-I）的效应量趋近于零。

**场独立/依存为什么作为可选？** Witkin 的场独立/场依存（FI/FD）确实在招聘领域有应用（飞行员选拔、建筑师筛选），但它与 Riding 的 W-A 维度存在中度相关（r≈0.4-0.5, Riding & Cheema, 1991），不是完全独立的第二个维度。Demo 阶段默认只采 W-A，FI/FD 作为"如果有多余设计精力就加上"的可选项。

**为什么用 -1.0 到 +1.0 而非 0-10？** Riding 的 CSA 原始输出是一个**反应时比率**，天然落在 -1 到 +1 之间（负值=整体型, 正值=分析型）。这个区间保留了"零点是自然中点（平衡型）"的语义（0 = 两种风格没有任何偏向）。转换为 0-10 会丢失零点的含义。在 Layer 2 的 `integrator.js` 中统一标准化为 [0,1] 进行跨游戏融合，但不改变原始输出的-1~+1 格式。

**为什么增加 `flexibility` 和 `impulseScore` 两个行为指标？** Kagan (1966) 的反思-冲动维度补充了 Riding CSA 的盲区——CSA 只测"你是哪种风格"，不测"你能多快切换到另一种风格"。`flexibility` 捕获认知灵活性（高低可逆的元认知能力），`impulseScore` 区分"你是分析型因为你深思熟虑"还是"你是分析型因为你害怕犯错"——两者行为相同但心理动因不同。

**EEG 证据的价值**：刘惠军 & 李洋 (2006) 的脑电研究为 W-A 维度提供了**生理层面的效度证据**——整体型与右半球 α 波优势相关，分析型与左半球 β 波优势相关。这意味着我们测量的不是一个问卷自陈的模糊构念，而是一个有神经基础的认知特质。

### G4: 意义建构取向

```
game4_output = {
  gameId: "game4",
  dimensions: {
    presence: number,    // 拥有意义感: 当前感知到的意义清晰度 (0-10)
    search: number,      // 寻求意义倾向: 主动探索意义的行为强度 (0-10)
  },
  behaviorSummary: {
    profileType: string,           // "安定型" | "探索型" | "成长型" | "游离型"
    meaningSource: string,         // 主要意义来源: "影响他人" | "创造" | "揭示真相" | "解决实际问题"
  }
}
```

**评分逻辑**：Presence 和 Search 是正交维度，可同时高/低。通过价值观冲突场景中的选择偏好累加计分，归一化至 0-10。

**理论来源**：Steger MLQ (2006); Schneider P-O Fit; 意义建构理论 (Meaning-making)

### G4 合理性说明

**为什么选 Steger MLQ 而非其他意义测量工具？** 在四个主要候选框架中，MLQ 是唯一同时测量"当前状态"和"动态倾向"的：

| 框架 | 测量内容 | 为什么不够 |
|------|---------|-----------|
| Crumbaugh & Maholick PIL (1964) | 生命目的感（单维） | 只测"有没有意义"，不测"找不找意义" |
| Antonovsky SOC (1987) | 心理一致感（可理解/可管理/有意义） | 更像心理健康指标而非职业取向指标 |
| Frankl 存在真空 | 缺乏意义感（临床概念） | 临床导向，不适合组织场景 |
| **Steger MLQ (2006)** | 拥有意义 + 寻求意义（正交双维） | ✅ 双维度精准区分存在感与探索欲 |

**为什么拥有和寻求是正交而非对立？** 这是 MLQ 最关键的测量学特性——Steger et al. (2006) 在开发 MLQ 时的验证性因子分析（CFA）显示两因子模型显著优于单因子模型（χ² 差值 p<0.001）。你可以同时"感到人生有意义"且"仍在寻找更多意义"（成长型），也可以两者都低（游离型）。这四种排列组合天然构成了四个有区分的意义取向类型，比单一数字有更强的诊断力。

**为什么 `meaningSource` 被纳入？** MLQ 本身只测量"程度"不测量"方向"。但 Schneider (1987) 的 P-O Fit 理论指出，意义来源与组织文化的匹配是人员留存的强预测因子（比单纯的"意义感高低"更直接）。纳入 `meaningSource` 让匹配引擎不仅仅匹配"你有多在乎意义"，而是匹配"你在乎的哪种意义"——这对 Layer 3 的岗位推荐至关重要。

**为什么 MLQ 只有 10 题但我们能在游戏里测量？** MLQ 的传统题项（如"我理解生命的意义"）高度抽象，直接移植到游戏中会破坏沉浸感。本设计用**价值观冲突场景**替代 Likert 题项——例如：

> 题目场景："两个 offer 都很好。A：工作内容你确信有意义，但团队氛围一般。B：团队很棒，但工作本身你觉得平淡。你更可能——"

选项映射逻辑：选 A → Presence+1, Search+0（你已有意义，不需要寻找）；选 B → Presence-1, Search+1（你在寻找意义，当前还不满足）。这种设计将自陈问卷的"我觉得"转化为决策行为的"我选择"，保持了 MLQ 的理论结构但改变了测量范式。

**文献**：Georgiou et al. (2020) 的 Gamified SJT 研究证明，情境判断测验可以有效地替代 Likert 量表来测量软技能构念，只要评分锚定逻辑与源量表的因子结构保持一致。

## 五、Layer 2 AI 肖像层 —— 两阶段生成策略 ✅ 已确认

### 决策 B：翻译 + 洞察双阶段

```
Layer 1 数据（14个指标）
        │
        ▼
┌─────────────────────────────┐
│ 阶段一：翻译（基础交付）      │  ← 必须完成
│ 每个维度 → 可读的自然语言描述  │
│ "你的成就动机得分 7/10，属于   │
│  较高水平..."                 │
└─────────────┬───────────────┘
              │
              ▼
┌─────────────────────────────┐
│ 阶段二：洞察（加分交付）      │  ← 尽力而为
│ 跨维度交叉解读               │
│ "成就动机高+亲和动机也不低     │
│  → 你可能想赢，但想带着团队    │
│     一起赢..."                │
│                              │
│ 若 AI 质量不可靠 → 降级为     │
│ 仅翻译模式，不强行洞察        │
└─────────────────────────────┘
```

### 阶段一：翻译（Translation）

| 输入 | 输出示例 |
|------|---------|
| G1: nAch=7, nPow=4, nAff=6 | "你的核心驱动力以成就和亲和为主——你在乎把事情做好，也在乎和谁一起做。权力对你来说不是首要追求。" |
| G2: TF=8, SV=7, AU=3 | "你最无法放弃的是专业深度和帮助他人的机会。相比之下，纯粹的自主权不是你坚守的底线。" |
| G3: W-A=0.6 | "你倾向于分析型思维——面对复杂信息时，你会先拆解再理解，而不是先看全局。" |
| G4: Presence=7, Search=5 | "你对当下的方向有比较清晰的感知，同时也没有停止探索更多可能性。" |

**实现方式**：规则模板填充。每个维度根据分数区间（0-3/4-6/7-10）匹配预设文案段，确保兜底稳定。

### 阶段二：洞察（Insight）

| 洞察类型 | 触发条件 | 示例 |
|---------|---------|------|
| 内部张力 | 同一游戏内两个维度差值 ≥3 分 | "你的成就动机远高于权力动机——你追求卓越是为了战胜自己的标准，而非战胜他人。" |
| 跨游戏矛盾 | 不同游戏揭示出看似冲突的倾向 | "你追求创作自由（G2），但认知风格却偏分析型（G3）——你不是天马行空的创作者，而是有结构地创造。" |
| 一致性确认 | 多个指标指向同一结论 | "三个维度同时指向'关系导向'——驱动你的是人（G1），你放不下的是帮助人的机会（G2），你的意义也来自影响他人（G4）。" |
| 盲点揭示 | 某个维度得分异常低 | "你几乎不在乎安全/稳定——在别人焦虑不确定性的时候，你可能已经在行动了。" |

**实现方式**：AI 生成（混元/DeepSeek）。提示词中明确要求"只有在有真正发现时才输出洞察，不要硬凑"。

**降级策略**：AI 调用失败或返回质量不合格时，报告仅包含阶段一的翻译内容——这仍然是一份可用的肖像报告。

### Layer 2 两阶段策略的合理性

**为什么先翻译后洞察，而不是一步到位？** 这是工程上的"渐进增强"模式：

1. **可降级性**：如果 AI 挂了（API 超时、Token 耗尽、模型幻觉），翻译层是纯规则驱动的，不受影响。用户至少能拿到一份"你的成就动机 7/10，属于较高水平"的准确报告——不是完美的，但是可用的。
2. **质量控制**：翻译层的输出可以用单元测试验证（"输入 nAch=7 → 输出包含'较高水平'"），洞察层的输出只能人工评估。分开两层意味着出问题的是洞察层（可关闭），而不是整个报告（不能关）。
3. **学术先例**：2026 年 arXiv 论文 *LLM + Fuzzy TOPSIS for Personnel Selection* 采用了类似的双轨架构——规则引擎提供基线评分，LLM 提供置信区间和解释。该论文的结论是：双轨的输出质量优于任一单轨，且两者不一致时恰好是"需要人工复核"的信号。

**为什么洞察类型选了这 4 类？** 每个类型对应心理学中的一种效应：

| 洞察类型 | 对应的心理学概念 | 为什么"被说中感"强 |
|---------|---------------|-----------------|
| 内部张力 | 认知失调 (Festinger, 1957) | 点出"你其实在和自己矛盾"是种高级共情 |
| 跨游戏矛盾 | 交叉验证 / 三角测量 | "同一个你，在不同场景下表现不同"比单一维度描述更具人格复杂性 |
| 一致性确认 | 趋同效度 | "三个独立测量指向同一结论"增强可信度 |
| 盲点揭示 | Johari 窗 — 盲区 | 说出"你自己可能没注意到的"是洞察的本质 |
