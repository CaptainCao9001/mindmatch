# MindMatch 项目说明书

## 一、设计流程与数据采集

基于投射技术，4个游戏采集行为数据，避免量表的"知道该选什么"偏差。

**G1 核心驱动力**：5个职场情境，记录选项选择+决策时长+鼠标停留。心理学依据：McClelland三动机理论。输出3维（nAch成就/nPow权力/nAff亲和，0-10→0-1标准化）。

**G2 职业锚**：16张卡片，3阶段取舍（8→4→2），记录选择顺序+停留时长。依据：Schein职业锚理论。输出8维（TF技术/GM管理/AU自主/SE安全/EC创业/SV服务/CH挑战/LS生活，0-10→0-1）。

**G3 认知风格**：8段记忆碎片，2阶段（🟢🟡🔴分组→排序），记录分组+排序权重。依据：MBTI维度+Witkin认知风格理论。输出1维（wholistAnalytic整体分析倾向，-1~+1→0-1）。

**G4 意义建构**：6个场景，双重反应（presence拥有意义感+search寻求倾向）。依据：Steger MLQ量表。输出2维（presence/search，0-10→0-1）。

**数据整合**：localStorage存储→integrator.js标准化为14维UnifiedProfile（未完成维度置null，匹配时填充0.5中间值）。

## 二、匹配算法

**两级双轨混合引擎**：

Level 1用户→6方向（系统建构者/深度解读者/创意塑造者/人际联结者/价值驱动者/赋能陪伴者），Level 2 Top方向→54职业变体（每方向9职业，delta偏移叠加到方向基线）。

双轨并行：线性加权（主）两阶段加权输出0-100分数；TOPSIS（辅）向量归一化计算贴近度定排名。融合策略：TOPSIS定排名，线性给分数。高斯核函数 `exp(-d²/2σ²)`（σ=0.20）连续可导无死区。

## 三、智能体（Agent v2）

**定位**：测评结果验证+校准+深挖工具。**硬墙机制**：4游戏未完成则拦截进入。

**4阶段交互**：结果共鸣（2-3轮）→深度挖掘（3-4轮）→现实校准（2-3轮）→行动地图（2-3轮）。3个工具：advance_phase推进阶段、save_collected保存补充信息、finish_conversation结束对话。

## 四、整体流程

```
index.html → G1→G2→G3→G4 → results.html（14维思维肖像）
   ├─ 肖像报告（translator.js生成文字描述）
   ├─ AI解读（DeepSeek API个性化洞察）
   ├─ 职业匹配（6方向+54职业排行）
   └─ CTA → career-guide.html → Agent v2深挖验证
```

## 五、技术架构（实际落实）

**前端**：纯HTML/CSS/JS + ES Modules，零构建。模块分四层：core（store/api/utils）、games（4游戏引擎+数据+策略）、layers（integrator/translator/matcher纯计算）、ui（portrait/charts/match-ui纯渲染）。可视化用ECharts雷达图。

**后端**：CloudBase静态托管 + SCF云函数。API通过SCF服务端代理调用，密钥存环境变量，前端零配置。Agent服务：agent-v2/main.mjs（端口8101，本地/SCF双模式自动切换）。

**AI**：DeepSeek V3主引擎+混元fallback，双模型保障可用性。

**容错**：null维度自动填充0.5；TOPSIS失败降级仅线性；全局版本号加URL参数防CDN缓存。

## 六、效果评估

**功能完整性**：4游戏可运行，14维画像可视化，54职业两级匹配，AI洞察生成，报告导出全流程。**技术鲁棒性**：双轨匹配交叉验证，null值自动填充，模型fallback。**待验证**：匹配精度需真实数据验证，AI洞察个性度需A/B测试。
