# MindMatch 项目说明书

## 一、设计流程与参考依据

**1. 游戏化测评设计**
- 基于投射技术（Projective Technique），用户在沉浸式场景中做选择，暴露深层特质
- 参考心理学行为实验范式，记录选择内容、决策时长、鼠标停留时长三类过程数据
- 4个游戏分别对应：核心驱动力（G1）、职业锚（G2）、认知风格（G3）、意义建构（G4）

**2. 14维画像框架**
- 整合McClelland需求动机理论（成就/亲和/权力）、Schein职业锚理论、Witkin认知风格理论
- 构建4游戏×14维度的评估体系，每个维度0-1标准化

**3. 两级匹配算法**
- Level 1：用户→6方向（系统建构者/深度解读者/创意塑造者/人际联结者/价值驱动者/赋能陪伴者）
- Level 2：Top方向→54职业变体
- 双轨混合引擎（线性加权主+TOPSIS辅），高斯核匹配（σ=0.20）

**4. AI解读层**
- 将结构化评分转化为第二人称视角的个性化洞察
- DeepSeek（主）+混元（备）双模型fallback机制

## 二、工具选择与技术架构

| 层级 | 选型 | 理由 |
|------|------|------|
| 前端 | 纯HTML/CSS/JS + ES Modules | 零构建依赖，模块化架构便于迭代 |
| 可视化 | ECharts | 雷达图开箱即用，中文文档完善 |
| 部署 | CloudBase静态托管 + SCF云函数 | 国内可访问，与腾讯技术生态对齐 |
| AI层 | DeepSeek V3（主）+混元 | 中文质量高且性价比高，双模型fallback保障可用性 |

**技术架构**：
- 数据流：游戏行为→localStorage→integrator.js（整合）→translator.js（翻译）→portrait.js（可视化）+matcher_extended.js（匹配）
- API调用：服务端代理（SCF云函数），密钥存储于环境变量，前端零配置
- 版本管理：全局版本号`__MM_VER__`+URL参数`?v=`防CDN缓存

## 三、关键迭代设计

| 版本 | 核心改进 | 决策依据 |
|------|---------|---------|
| v1.0 | 4游戏+基础匹配 | 功能完整性优先 |
| v2.0 | Agent v2架构（4阶段交互）| 发现用户需要"验证+校准"而非一次性报告 |
| v20260611a | 浮点数精度修复+量表可视化 | 发现数值显示问题和进度条不可见bug |

**关键踩坑与解决**：
- CloudBase CDN缓存365天→所有模块引用加`?v=`版本号
- tcb deploy静默跳过→先delete再deploy
- 前端动态import失败→改用内联localStorage读取

## 四、效果评估

**功能完整性**：4游戏可完整运行，14维画像可视化，54职业两级匹配，AI洞察生成，报告导出（PDF/图片）全流程打通。

**技术鲁棒性**：双轨匹配引擎（线性+TOPSIS）交叉验证，null值自动填充防崩溃，DeepSeek/混元fallback保障AI层可用性。

**用户体验**：行为触发式反馈，进度条实时可见，移动端适配，导出报告支持离线保存。

**待验证项**：匹配算法精度需真实用户数据验证；AI洞察个性化程度需A/B测试优化。
