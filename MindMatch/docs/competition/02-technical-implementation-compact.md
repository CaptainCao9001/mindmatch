# 二、技术实现与方法参考

## 2.1 数据选用与采集

4 款游戏覆盖"动机—价值观—认知—意义"四层构念，基于 McClland 三动机理论、Schein 八锚模型、Riding CSA 认知风格、Steger MLQ 意义量表设计，采集行为过程量（决策时间、卡片操作序列、反应时差等）。4 款游戏输出汇聚为 14 维标准化画像，同时保留全部过程量供 AI 深度解读。设计参考 MTMM（多特质多方法矩阵）。

## 2.2 匹配算法

双轨混合决策：

**轨道一（线性加权）**：高斯核函数（$\sigma=0.20$）连续计算匹配度，消除阶跃函数决策死区。两阶段加权——先按游戏内维度聚合，再按游戏权重合成总分。

**轨道二（TOPSIS 多准则决策）**：6 步标准流程——矩阵构建 → 归一化 → 加权 → 理想解 → 距离计算 → 贴近度排序。

**双轨融合**：两条轨道结果交叉验证，经对比度拉伸（K=1.4）放大方向间区分度。

**两级匹配**：第一级匹配 6 大职业方向，第二级在 Top 3 方向内匹配 54 个具体职业。

## 2.3 智能体对话

Agent v2 基于 Function Calling 实现结构化对话编排。

**输入**：14 维画像 + 行为叙事 + 方向排名 + 用户消息
**输出**：结构化回复 + 阶段推进信号

**4 阶段设计**：
- 结果共鸣（讨论测评结果，2-3 轮）
- 深度挖掘（追问矛盾点，3-4 轮）
- 现实校准（关联实际处境，2-3 轮）
- 行动地图（生成验证实验，2-3 轮）

DeepSeek 为主，失败时自动切换混元。
