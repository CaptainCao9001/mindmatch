## 二、设计理念：投射式测评

与传统"选描述句"的职业锚问卷不同，G2 采用**投射技术**：

- 用户不知道自己在测什么 → 降低社会赞许性偏差
- 每张卡片是学生时代的"记忆碎片"而非"锚定义" → 真实偏好自然流露
- 三阶段递进加深 → 越到后面越接近核心需求

核心公式：**情境代入 → 直觉选择 → 锚定提取**

## 三、三阶段交互流程

```
阶段一「百团大战」→ 反馈一 → 阶段二「大三抉择」→ 反馈二 → 阶段三「毕业拍立得」→ 完成页
    16→8 张           8→3~4 张          3~4→1~2 张
```

### 阶段一「百团大战」— 大一 · 操场招新（16→8）

**场景设定**：开学季，操场上搭了一百多个帐篷，学长学姐们举着海报朝你招手。你只能选一半加入。

**操作**：翻牌查看详情 → 「加入」或「路过」

**视觉风格**：社团传单/海报风格，每张卡片正面是社团名+口号，翻牌后显示详细描述。加入的卡片飞入"书包"保留区。

### 阶段二「大三抉择」— 精力有限（8→3~4）

**场景设定**：到了大三。课表满了，实习、论文、考证挤占时间。书包里的社团，你必须做出取舍。

**操作**：对每张卡片做「继续投入」或「遗憾退出」，纯凭喜好。

**视觉风格**：卡片以"课表条目"形式排列，退出的卡片变灰→缩小→飞出屏幕。

### 阶段三「毕业拍立得」— 最终确认（3~4→1~2）

**场景设定**：毕业了。宿舍开始收拾行李。翻开手机相册，几百张大学照片。只能留住 1~2 张，你选哪张？

**操作**：点选 1~2 张最触动自己的照片。

**视觉风格**：拍立得照片风格（白边、微倾斜、暖光滤镜）。选中的照片"亮起来"，其余渐变黑白+模糊。选择完成后照片装进"行李箱"。

## 七、数据计算逻辑

### 得分公式

```javascript
// 每个锚的得分 = 阶段权重 × 是否被选中
const STAGE_WEIGHTS = [1, 2, 3]; // 阶段一/二/三的权重

// 单张卡片对该锚的贡献
function calcAnchorScore(cards, stages) {
  let score = 0;
  for (const card of cards) {
    if (card.selectedInStage >= 0) {
      score += STAGE_WEIGHTS[card.selectedInStage];
    }
  }
  return score;
}
```

### 权重设计理由

- 阶段一（权重 1）：广撒网阶段，选择较随意
- 阶段二（权重 2）：开始取舍，选择更有意义
- 阶段三（权重 3）：最终确认，选择最核心

### 原始分范围

每个锚：0~9 分（两阶段均选满分：1+2+3+1+2+3=12，但同一阶段最多 1 张/锚被选，实际范围 0~6）
归一化到 0-10 用于结果页展示。

## 十、UX 设计要点

### 10.1 视觉风格

- **整体色调**：暖黄色调（大学回忆感）+ 旧照片质感滤镜
- **阶段一**：传单/海报风格，色彩丰富、青春感
- **阶段二**：课表风格，简洁有序，轻微紧迫感
- **阶段三**：拍立得风格，暖光、白边、微倾斜、怀旧滤镜

### 10.2 动画设计

| 动作 | 动画 | 时长 |
|------|------|------|
| 翻牌 | 卡片 3D 翻转 180° | 400ms |
| 加入 | 卡片飞入右侧"书包" | 500ms |
| 路过 | 卡片向左滑出 + 缩小 | 400ms |
| 丢弃（阶段二） | 卡片变灰 → 缩小 → 飞出 | 600ms |
| 照片选中（阶段三） | 亮起 + 轻微放大 | 500ms |
| 照片未选中 | 渐变黑白 + 模糊 | 800ms |
| 装进行李箱 | 照片飞入行李箱图标 | 600ms |

### 10.3 无障碍

- 翻牌：键盘 Tab 导航 + Enter 翻牌 + 左右箭头选择
- 拍立得：Space 选中/取消
- aria-label 描述每张卡片内容
- 减弱动画：prefers-reduced-motion 时仅用淡入淡出
