MindMatch 项目技术文档

一、问题诊断与方案设计

1.1 问题诊断

根据近年权威数据显示，当代中国年轻人面临职业选择困境：73% 大学生困惑方向，44% 毕业生后悔选专业，35% 学非所用，48% 跳槽后悔，职场满意度仅 2.72/5。这一现象背后是自我认知的普遍缺失——导致职业决策带有盲目性。

现有测评工具（MBTI、霍兰德等）依赖自陈量表，存在社会赞许性偏差，且仅输出标签，无法捕捉真实行为模式，难以为职业决策提供充分依据。

1.2 方案设计

目标人群：处于职业规划窗口期的在校大学生，以及有转行探索需求的在职人员。

使用场景：校招批量初筛方向匹配、企业内部转岗评估、职业咨询师辅助工具。

MindMatch 通过 4 款小游戏采集心理特质及决策过程量，匹配算法聚焦大方向，智能体结合多源数据引导用户自省，给出更准确的职业方向建议。

设计定位：基于心理学框架，通过游戏化过程数据采集行为模式，结合大模型多源数据，引导用户自我探索、验证职业方向并提供可执行的行动建议。

4 款游戏采集 → 14 维画像 + 行为叙事 → 肖像页 + 职业匹配 → Agent 对话

核心原则：通过心理学和数据指引，帮用户自我探索。

核心创新：多源数据融合（14 维画像 + 行为叙事）+ AI 引导式对话，了解用户实际情况，进行动态校准与建议。

二、技术实现

2.1 数据选用与采集

4 款游戏覆盖"动机—价值观—认知—意义"四层构念，基于 McClelland 三动机理论、Schein 八锚模型、Riding CSA 认知风格、Steger MLQ 意义量表设计，采集行为过程量（决策时间、卡片操作序列、反应时差等），输出汇聚为 14 维标准化画像，保留过程量供 AI 结合多源数据生成更准确建议。选用这四类游戏，覆盖职业选择中"想做什么（动机）— 适合什么（价值观）— 怎么思考（认知）— 为何坚持（意义）"四个决策维度，避免单维度误判。

2.2 匹配算法

双轨混合：轨道一用高斯核（σ=0.20）消除阶跃函数决策死区，两阶段加权；轨道二用 TOPSIS 多准则决策。双轨交叉验证后对比度拉伸（K=1.4），两级匹配（6 方向→54 职业）。

2.3 智能体对话

Agent v2 基于 Function Calling 实现结构化编排，融合 14 维画像、行为叙事、方向排名等多源数据引导用户深度自省，输出更准确的职业方向建议。4 阶段：结果共鸣（2-3 轮）→ 深度挖掘（3-4 轮）→ 现实校准（2-3 轮）→ 行动地图（2-3 轮）。

2.4 技术架构

基于 HTML/CSS/JS（ES Modules）+ 腾讯云 CloudBase 静态托管 + SCF 云函数 + DeepSeek/混元 API 构建。

三、关键迭代过程

迭代一：匹配算法

  • 迭代前：阶跃函数硬截断，存在决策死区；单一算法路径，无交叉验证
  • 迭代后：高斯核（σ=0.20）连续可导消除死区；TOPSIS 双轨混合交叉验证；两级匹配（6 方向 → 54 职业）

迭代二：Agent 架构

  • 迭代前：v1 五阶段独立对话，状态分散，工具全集常驻，易出现跳阶和幻觉
  • 迭代后：v2 四阶段工具编排，统一状态机，动态工具枚举（每阶段仅暴露允许工具），加入 ensureEndsWithQuestion 后处理器强制提问

四、效果评估

系统已完成端到端流程验证，匹配逻辑与 Agent 对话流程符合设计预期。当前 demo 仅供参考，匹配准确度需更多真实用户数据持续校准优化。
