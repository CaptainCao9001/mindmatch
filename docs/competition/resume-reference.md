# MindMatch 项目详细描述（简历参考用）

---

## 一、项目概述

MindMatch 是一款 **AI 驱动的职业方向测评 Web 应用**。用户通过 4 款游戏化测评完成心理特质和行为过程数据的采集，系统生成 14 维人格画像，经过双轨匹配算法排序 6 大方向 × 54 个职业，最后通过 AI Agent 四阶段对话进行个性化解读与行动建议。全栈独立开发，部署于腾讯云 CloudBase。

- **项目类型**：个人独立项目
- **开发周期**：2026.03 – 2026.06（约 3 个月）
- **线上地址**：https://mindmatch-d0gz847n4e29e3181-1438477634.tcloudbaseapp.com
- **GitHub**：https://github.com/CaptainCao9001/mindmatch
- **代码规模**：50+ 源文件，纯 HTML/CSS/JS（ES Modules），零构建工具

## 二、问题诊断

### 2.1 背景数据

| 指标 | 数据 | 来源 |
|------|------|------|
| 大学生困惑人生方向 | 73.39% | 中国青年报 2023（478 所高校，10,095 份问卷） |
| 毕业生后悔选专业 | 44% | 美联储 SHED 调查 |
| 高校毕业生学非所用 | 35% | 中国就业蓝皮书 |
| 跳槽者后悔跳槽 | 47.86% | 前程无忧（6,000 人） |
| 中国职场满意度 | 2.72/5 | 智联招聘 2024 |

### 2.2 核心判断

这一数据链条指向一个深层问题：**年轻人可能选错职业，根源是自我认知的普遍缺失**——个体缺乏对自身真实特质的有效认知，职业决策带有盲目性。

现有测评工具（MBTI、霍兰德等）的两大缺陷：
1. **自陈量表偏差**：用户可刻意选择"社会期望"答案，无法捕捉真实行为模式
2. **输出单一**：仅给标签，缺乏结合个人实际情况的解读和行动建议

## 三、方案设计

### 3.1 目标人群与场景

| 维度 | 说明 |
|------|------|
| 目标用户 | 职业规划窗口期的在校大学生；有转行探索需求的在职人员 |
| 使用场景 | 校招批量初筛方向匹配；企业内部转岗评估；职业咨询师辅助工具 |

### 3.2 系统流程

```
4 款游戏采集 → 14 维画像 + 行为叙事 → 肖像页 + 职业匹配 → Agent 对话
   心理特质         积分器融合           双轨并行          4 阶段校准
   决策过程量        MTMM 交叉验证        雷达图            共鸣→深挖
                                       6 方向×54 职业      现实→行动
```

### 3.3 设计定位

**基于心理学框架，通过游戏化过程数据采集行为模式，结合大模型多源数据，引导用户自我探索、验证职业方向并提供可执行的行动建议。**

### 3.4 核心原则

**通过心理学和数据指引，帮用户自我探索**——用户不是"被归类"，而是通过自己真实的行为数据认识自己，通过 AI 结合多源数据得到更准确的职业方向建议。

### 3.5 核心创新

**多源数据融合（14 维画像 + 行为叙事）+ AI 引导式对话**，了解用户实际情况，进行动态校准与建议。与传统测评工具的差异：不只输出标签，而是结合用户在测评过程中的真实行为（决策速度、选择序列、翻看次数等），用大模型进行个性化解读。

## 四、技术细节

### 4.1 数据采集（4 款游戏）

| 游戏 | 理论框架 | 采集内容 | 过程量 | 输出维度 |
|------|---------|---------|--------|:--:|
| G1 核心驱动力 | McClelland 三动机理论 (1961) | 5 个职场情境选择（SJT 格式） | 反应时间（ms）、场景序列 | 3 维（nAch/nPow/nAff） |
| G2 职业锚 | Schein 八锚理论 (1978) | 16 张卡片自由保留/丢弃 | 卡片翻看次数、决策序列 | 8 维（8 个职业锚） |
| G3 认知风格 | Riding CSA (1991) | 8 段记忆碎片自由排序 | 排序耗时、操作次数（impulse） | 1 维（分析型↔整体型） |
| G4 意义建构 | Steger MLQ (2006) | 6 场景双反应（感受+行动） | 两轮反应时差（ms） | 2 维（presence/search） |

4 款游戏形成"动机—价值观—认知—意义"四层构念覆盖，对应职业选择中"想做什么→适合什么→怎么思考→为何坚持"四个决策维度，避免单维度误判。最终汇聚为 14 维标准化画像，并保留全过程量（反应时间、决策序列、翻看次数、排序节奏等），供行为叙事层和 AI 解读使用。设计参考 MTMM（多特质多方法矩阵），通过不同测量方法交叉验证同一心理构念。

### 4.2 匹配算法

**双轨混合决策引擎**：

**轨道一：线性加权 + 高斯核**
- 核函数：$\text{match}(\text{actual}, \text{ideal}) = \exp\left(-\frac{(\text{actual} - \text{ideal})^2}{2\sigma^2}\right),\ \sigma=0.20$
- 选用理由：相比阶跃函数的硬截断（阈值边界处微小差异导致匹配度剧烈变化），高斯核连续可导，无决策死区。σ=0.20 意味着距离理想 0.2 时匹配度约 0.61，距离 0.4 时降至约 0.14。
- 两阶段加权架构：Phase 1 游戏内按维度聚合 → Phase 2 游戏间加权合成，消除"维度数量不平衡"（如 G2 有 8 维、G3 仅 1 维）

**轨道二：TOPSIS 多准则决策**
- 经典 6 步流程：矩阵构建 → 归一化 → 加权 → 理想解确定 → 欧氏距离计算 → 贴近度排序
- 从"整体剖面距离"角度评估，对维度间协调性更敏感，弥补线性加权对异常维度不敏感的缺陷

**双轨融合与后处理**：
- 双轨交叉验证：线性 Top1 ≠ TOPSIS Top1 或排名差异 ≥ 2 → 触发 AI 审核标记
- 对比度拉伸：`stretched = mean + (score - mean) × 1.4`，clamp `[0, 100]`，放大方向间视觉差异
- 两级匹配：第一级匹配 6 大方向，第二级在 Top 3 方向内匹配 54 个职业
- 职业剖面采用"基线 + Delta"复用设计：6 方向定义基线，54 职业仅存储相对基线的 delta（约 7 个非零值/职业）

### 4.3 Agent 对话系统

**v2 架构（4 阶段工具编排）**：

| 阶段 | 目标 | 轮次 | 说明 |
|------|------|:--:|------|
| Phase 1 结果共鸣 | 讨论测评结果，建立信任 | 2-3 轮 | 先让用户认可数据，再引导思考 |
| Phase 2 深度挖掘 | 追问矛盾点和关键特质 | 3-4 轮 | 检测画像中的冲突信号（如高 nAch + 高 nAff 的张力） |
| Phase 3 现实校准 | 关联用户实际处境 | 2-3 轮 | 将心理画像落实到"你现在的工作/学习是什么样" |
| Phase 4 行动地图 | 生成可执行的验证实验 | 2-3 轮 | 给出具体下一步行动建议 |

**技术实现**：
- 基于 Function Calling，3 工具注册表模式（advance_phase / save_collected / finish_conversation）
- 动态工具枚举：每阶段仅暴露该阶段允许的工具，防止 LLM 跳阶
- 统一状态机管理阶段推进，加入 `ensureEndsWithQuestion` 后处理器强制提问
- DeepSeek 优先，失败自动切换混元 Fallback

### 4.4 技术架构

| 层 | 技术选型 | 说明 |
|----|---------|------|
| 前端 | HTML/CSS/JS + ES Modules | 50+ 文件按 `core / games / layers / ui` 四层架构，零构建工具 |
| 数据桥梁 | localStorage | 唯一的数据持久化方式，无需后端数据库 |
| AI 代理 | Node.js SCF 云函数 | 服务端代理 DeepSeek/混元 API，密钥环境变量管理，客户端零配置 |
| 静态托管 | 腾讯云 CloudBase + CDN | 一键部署，自动 HTTPS |
| 算法 | JavaScript 原生实现 | 高斯核、TOPSIS、对比度拉伸等全部手写，无外部算法库依赖 |

**前端分层架构**：
- `js/core/` — 基础设施（store/api/utils），无 UI
- `js/games/` — 4 款游戏引擎 + 数据 + 策略，无结果页逻辑
- `js/layers/` — 纯数据计算与翻译，零 DOM
- `js/ui/` — 纯渲染，零业务逻辑

## 五、关键迭代过程

### 迭代一：匹配算法升级

| | 迭代前 | 迭代后 |
|------|---------|---------|
| 核函数 | 阶跃函数硬截断，存在决策死区 | 高斯核（σ=0.20）连续可导，平滑衰减 |
| 算法路径 | 单一线性加权，无交叉验证 | TOPSIS 双轨混合，交叉验证 |
| 匹配粒度 | 仅方向级 | 两级匹配（6 方向 → 54 职业） |
| 后处理 | 原始分数直接展示 | 对比度拉伸（K=1.4）放大区分度 |

### 迭代二：Agent 架构重构

| | 迭代前（v1） | 迭代后（v2） |
|------|---------|---------|
| 阶段结构 | 5 阶段独立对话，状态分散 | 4 阶段统一状态机，结构化编排 |
| 工具策略 | 工具全集常驻，易跳阶和幻觉 | 动态工具枚举，每阶段仅暴露允许工具 |
| 对话质量 | 回复可能以陈述结束，不引导下一步 | 加入 ensureEndsWithQuestion 后处理器，强制提问 |
| 吞吐量 | 未做优化，多轮对话重复消耗 token | 工具注册表模式减少冗余，节省 token |
