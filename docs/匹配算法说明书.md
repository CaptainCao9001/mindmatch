## 三、匹配算法核心：两级双轨引擎

### 3.1 整体架构

```
用户 14 维标准化向量
        │
        ├──────────────────┬──────────────────┐
        ▼                  ▼                  ▼
   轨道一：线性加权    轨道二：TOPSIS     AI 审核（条件触发）
   (主分数)           (定排名)          (双轨不一致时)
        │                  │                  │
        └──────────────────┼──────────────────┘
                           ▼
              Level 1: 6 方向排名
              Level 2: Top3 方向 → 54 职业排名
```

### 3.2 高斯核匹配函数（两轨道共用）

```
match(actual, ideal) = exp(-(actual - ideal)² / (2σ²))

参数: σ = 0.20（带宽/宽容度）
```

**为什么用高斯核而非阶跃函数？**

- 连续可导：用户与理想值偏差 0.01 和 0.02 的匹配度有细微差异，而非一刀切
- 无死区：阶跃函数在阈值处产生不连续跳变，高斯核提供平滑过渡
- 心理学依据：人格特质与岗位要求之间不存在硬边界，"足够接近"本身就是连续的

**σ 参数含义**：σ 越小越严格。σ=0.20 表示当用户值与理想值相差 0.2 时，匹配度降至约 exp(-0.5)≈0.61；相差 0.4 时降至 exp(-2)≈0.14。

## 四、两阶段加权（线性轨道）

### Phase 1：游戏内维度加权

```
gameScore(gameId) = Σ( match(dim, ideal) × weight ) / Σ(weight )
```

- 对每个游戏的若干维度，用高斯核计算各维度匹配度
- 按 `profiles[].weight` 加权平均，得到 4 个游戏得分（G1/G2/G3/G4，各 [0,1]）

**权重设计依据**：权重反映该维度对职业/方向的重要性。例如，系统建构者方向 TF(技术)权重 0.30 最高，因为技术能力是架构师的核心要求。

### Phase 2：游戏间加权

```
finalScore = G1_score × wG1 + G2_score × wG2 + G3_score × wG3 + G4_score × wG4
```

- 各方向/职业有独立 `gameWeights`（默认各 25%）
- 最终得分 × 100 → [0, 100] 展示分

**gameWeights 设计依据**：不同职业对四类能力的依赖不同。例如：
- **系统建构者**：G3 认知风格权重 0.35（逻辑分析是核心）
- **人际联结者**：G1 动机权重 0.30（亲和动机是关键）
- **创意塑造者**：G4 意义建构权重 0.30（寻求意义驱动创造）

## 五、TOPSIS 轨道（定排名）

### 5.1 算法流程（7 步）

| 步骤 | 操作 | 公式 |
|------|------|------|
| **Step 1** | 构建决策矩阵 D[m×n] | D[i][j] = gaussianMatch(user[j], ideal[i][j], σ=0.20) |
| **Step 2** | 向量归一化 | R[i][j] = D[i][j] / √(Σ D[k][j]²) |
| **Step 3** | 加权 | V[i][j] = R[i][j] × profiles[i][j].weight |
| **Step 4** | 正/负理想解 | A⁺[j] = max(V[i][j]), A⁻[j] = min(V[i][j]) |
| **Step 5** | 欧氏距离 | d⁺[i] = √(Σ (V[i][j] - A⁺[j])²), d⁻[i] = √(Σ (V[i][j] - A⁻[j])²) |
| **Step 6** | 贴近度 | C[i] = d⁻[i] / (d⁺[i] + d⁻[i]) |
| **Step 7** | 按 C 降序排 | — |

### 5.2 TOPSIS 为什么适合职业匹配

- **可解释**：正理想解 = "完美匹配该职业的候选人剖面"，每个用户的贴近度是量化的"距离完美有多远"
- **无标度影响**：向量归一化消除不同维度数值范围的差异
- **多准则**：14 个维度同时参与决策，而非单独打分后简单求和
- **学术验证**：2024 年 MDPI 研究在 5 种 MCDM 方法中验证 TOPSIS 在人事选拔中表现最优

## 六、双轨融合 + 对比度拉伸

### 6.1 融合策略

```
TOPSIS 定排名 → 线性定分数
```

| 轨道 | 用途 | 输出 |
|------|------|------|
| 线性加权 | 产生 0-100 展示分数 | 用户友好、直观 |
| TOPSIS | 产生排名顺序 | 数学严谨、多维均衡 |

### 6.2 对比度拉伸（方向级）

```
stretched = mean + (rawScore - mean) × K
K = 1.4（对比度系数）
最终 clamp 到 [0, 100]
```

**为什么要拉伸？** 6 个方向的得分常集中在 40-60 区间，用户难以区分。拉伸以均值为中心，放大偏差（K>1），使排序差异更明显，同时保持排名顺序不变。

## 七、两级匹配

### Level 1：用户 → 6 方向

| 方向 | 含义 | 关键维度偏好 |
|------|------|-------------|
| 系统建构者 | 架构、逻辑、系统思维 | TF↑, wholistAnalytic↑, nAff↓ |
| 深度解读者 | 分析、洞察、理论钻研 | wholistAnalytic↑, TF↑, nPow↓ |
| 创意塑造者 | 想象、表达、创新突破 | EC↑, search↑, SE↓ |
| 人际联结者 | 沟通、共情、关系构建 | nAff↑, SV↑, presence↑ |
| 价值驱动者 | 目标、成就、影响力驱动 | nAch↑, nPow↑, GM↑ |
| 赋能陪伴者 | 支持、培养、服务他人 | SV↑, nAff↑, nPow↓ |

### Level 2：Top 3 方向 → 54 职业

- 每个方向 9 个职业变体
- 职业剖面 = 方向基线 + delta 偏移

```
jobProfile.ideal[dim] = clamp(directionProfile.ideal[dim] + delta[dim], 0, 1)
```

**delta 设计依据**（Cohen's d 效应量）：
- |delta| > 0.15 为中等效应：同一方向下职业间的显著差异
- |delta| ≤ 0.10 为小效应：方向内子职业的微调
- 非零 delta 占比 50%（378/756），避免过度区分

## 八、AI 审核机制

### 8.1 触发条件

满足以下任一条件时，标记需要 AI 审核：

1. **线性 Top1 ≠ TOPSIS Top1**：两种算法对最优解的判断不一致
2. **某岗位双轨排名差 ≥ 2**：两种算法对某个岗位的评价存在显著分歧
3. **TOPSIS Top1 贴近度 < 0.6**：没有明显最优解，用户剖面与所有岗位都较远

### 8.2 AI 审核设计模式

参考 LLM + Fuzzy TOPSIS 论文（arXiv 2601.22433）：AI 不替代算法，而是作为"第二意见"——在算法出现分歧时提供置信度评估和理由解释。正常情况（双轨一致 + 明显最优）下 AI 不介入。

## 九、容错设计

| 场景 | 策略 |
|------|------|
| **未完成游戏（维度 null）** | 填充 0.5（中间值），确保匹配引擎不崩溃 |
| **TOPSIS 计算失败** | 降级为仅线性排名（`try/catch` 包裹） |
| **directionRanking 为空** | 返回空数组 + 日志告警 |
| **jobVariants 为空** | 跳过该方向，不展开职业列表 |
| **浮点数精度** | `Math.round(... * 10000) / 10000` 统一 4 位精度 |
