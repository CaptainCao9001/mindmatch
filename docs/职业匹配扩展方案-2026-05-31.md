## 一、MindMatch 14 维 → O*NET 维度映射方法论

当前 `ideal-profiles.json` 的 14 个 `field`：

| # | MindMatch 维度 | 含义 | O*NET 对应维度 |
|---|---|---|---|
| 1 | `nAch` | 成就驱动 | Work Values: Achievement |
| 2 | `nPow` | 权力驱动 | Work Values: Authority + Leadership |
| 3 | `nAff` | 亲和驱动 | Work Values: Relationships + Social |
| 4 | `TF` | 技术职能 | Abilities: Technical Design + Technology Design |
| 5 | `GM` | 常规管理 | Work Styles: Leadership + Self-Control |
| 6 | `AU` | 自主性 | Work Values: Independence + Autonomy |
| 7 | `SE` | 安全感 | Work Values: Support + Working Conditions |
| 8 | `EC` | 创业欲 | Work Values: Entrepreneurship + Initiative |
| 9 | `SV` | 服务价值 | Work Values: Help Others + Social Service |
| 10 | `CH` | 挑战欲 | Work Styles: Innovation + Adaptability |
| 11 | `LS` | 生活方式 | Work Values: Work-Life Balance + Flexibility |
| 12 | `wholistAnalytic` | 整体/分析认知 | Abilities: Deductive Reasoning + Inductive Reasoning |
| 13 | `presence` | 在场感（被看见） | Work Styles: Social Orientation + Persuasion |
| 14 | `search` | 探索欲 | Work Styles: Achievement/Effort + Initiative |

### 映射规则（草案）

O*NET 每个维度有 1-5 分的量表，转换为 [0, 1] 区间：

```
MindMatch ideal 值 = (O*NET 评分 - 1) / 4
方向 high → ideal ∈ [0.65, 0.95]
方向 mid  → ideal ∈ [0.35, 0.65]
方向 low  → ideal ∈ [0.05, 0.35]
```

## 六、算法匹配方案 — 架构设计（2026-05-31 追加）

### 6.1 核心算法决策

| 决策 | 结论 |
|------|------|
| 匹配层级 | **两级匹配** — 第一级 6 方向（TOPSIS + 线性双轨），第二级具体职业（线性单轨） |
| 匹配函数 | **分区匹配不变** — `mapSubDimension(actual, direction)` 三段线性，不改为距离匹配 |
| 职业建模 | **原型 + delta** — 方向 = 完整 14 维剖面，职业 = 方向基线 + delta 偏移（仅改 2-4 维） |
| 引擎策略 | **完全复用** — `matcher_linear.js` / `matcher_topsis.js` 不改，新增 `matcher_extended.js` |
| 兼容策略 | **URL 参数切换** — `match.html?mode=extended` 启用新模式，默认 v1 不变 |
| 展示交互 | **Top 1 展开 + 其余折叠** — 其余方向点击懒加载职业列表 |

```json

{
  "_version": "2.0.0",

    "system-builder": { /* DirectionProfile — 完整 14 维剖面 */ },

    "creative-molder": { /* ... */ },
    "people-connector": { /* ... */ },
    "value-driver": { /* ... */ },
    "empowerment-companion": { /* ... */ }
  },
  
  "directionJobs": {
    "system-builder": [
      { "id": "software-engineer", "name": "软件工程师", "delta": { "TF": 0.05, "AU": 0.05 } }
      // ... 每个方向 9 个职业
    ]
  },
  
  "jobs": { /* 保留 v1 4 岗位，不变 */ }

```

**delta 合并规则**：`职业实际 ideal = 方向基线 ideal + delta（clamped [0,1]）`，direction 和 weight 从方向原型继承。

### 6.4 两级匹配数据流

```
用户 14 维剖面 → matchExtended()
  │
  ├─ [Level 1] directionMatch() → TOPSIS + Linear 双轨 → 6 方向排名
  │     DirectionMatchEntry[] { directionId, name, score, closeness, rank, breakdown }
  │
  └─ [Level 2] jobMatch() → resolveJobProfile(基线+delta) → 线性匹配 → 9 职业排名
        JobMatchEntry[] { jobId, name, score, dimDetails }
  │
  └─ 融合 → ExtendedMatchReport { directionRanking[], needsAIReview }
```
