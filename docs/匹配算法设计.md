## 一、设计原则

| 原则 | 说明 |
|------|------|
| **肖像与匹配分离** | 管道 A（AI 肖像报告）和管道 B（岗位匹配引擎）共享输入数据但独立运行，互不依赖 |
| **规则兜底** | 规则引擎负责核心评分——透明、可解释、零外部依赖 |
| **AI 加分** | AI 提供补充视角（置信区间、推荐理由），失败不影响基础匹配结果 |
| **合成数据集驱动** | 算法调参和验证全部通过虚拟数据完成，不依赖真实用户测试 |

## 四、算法方案对比

### 4.1 学术文献综述

| 模型 | 来源 | 核心思路 | 与本项目的关联 |
|------|------|---------|-------------|
| **TOPSIS** | MDPI Mathematics (2024) | 多准则决策：计算候选人与理想解的距离 | ⭐ 直接可用——每个岗位定义一个理想剖面，计算用户到各理想剖面的距离 |
| **PJFNN** | arXiv 1812.08947 (2018) | CNN双塔：分别编码人才和岗位，再匹配 | 概念参考——"用户剖面" vs "岗位剖面"的双塔结构 |
| **PJFFF** | CSDN综述 (2023) | 显式特征(语义实体) + 隐式特征(行为序列)融合 | 参考其"多特征融合"思路：我们融合游戏心理数据 + 行为数据 |
| **LLM + Fuzzy TOPSIS** | arXiv 2601.22433 (2026) | LLM提取特征 + 模糊TOPSIS排序 | ⭐ AI辅助评分 + 置信区间的设计模式 |
| **Gallup人格匹配** | MDPI Algorithms (2025) | Gallup测试提取人格特征 → 岗位推荐 | 参考人格→岗位的映射方式 |

### 4.2 三种可选方案

| 方案 | 算法 | 复杂度 | 可解释性 | 适用场景 |
|------|------|:---:|:---:|---------|
| **A: 加权线性评分** | `Σ(维得分 × 权)` | 低 | ⭐⭐⭐ 极高 | Demo 底线方案 |
| **B: TOPSIS 距离法** | 计算用户到4个理想剖面的欧氏距离 | 中 | ⭐⭐ 较高 | 推荐采用 |
| **C: 余弦相似度** | `cos(用户向量, 岗位向量)` | 低 | ⭐⭐ 较高 | 备选/对比 |

### 4.3 推荐方案：B（TOPSIS）+ A（线性加权）混合

**理由**：
- TOPSIS 是人事选拔领域最成熟的 MCDM 方法（2024年 MDPI 对比研究验证）
- 线性加权作为"基准线"——当 TOPSIS 结果和线性结果一致时，增强可信度；不一致时触发 AI 审核
- 两种方法计算成本都极低（纯 JS 数毫秒），适合 Demo

## 五、推荐算法：双轨混合匹配引擎

### 5.1 整体流程

```
用户 profile (14维标准化向量)
        │
        ├──────────────────┬──────────────────┐
        ▼                  ▼                  ▼
   规则引擎（必须）      TOPSIS（必须）     AI 评估（加分）
        │                  │                  │
        ▼                  ▼                  ▼
  线性加权得分        理想解距离+贴近度    LLM置信区间+理由
        │                  │                  │
        └──────────────────┼──────────────────┘
                           ▼
                    最终排序 + 推荐
```

### 5.2 轨道一：线性加权（规则引擎）

#### 步骤 1：子维度 → 岗位映射（分段线性）

基于"四大方向特征提取.md"中的权重框架，对每个用户维度执行分段线性映射：

```javascript
function mapSubDimension(actualScore, direction, weight) {
  // actualScore: 标准化后的 [0,1] 值
  // direction: "high" | "low" | "mid"
  
  let matchScore;
  if (direction === "high") {
    if (actualScore >= 0.7) matchScore = 1.0;
    else if (actualScore >= 0.4) matchScore = (actualScore - 0.4) / 0.3;
    else matchScore = 0;
  } else if (direction === "low") {
    if (actualScore <= 0.3) matchScore = 1.0;
    else if (actualScore <= 0.6) matchScore = (0.6 - actualScore) / 0.3;
    else matchScore = 0;
  } else { // "mid"
    matchScore = 1.0 - Math.abs(actualScore - 0.5) * 2;
  }
  
  return matchScore * weight;
}
```

#### 步骤 2：游戏级加权聚合

```javascript
function gameLevelScore(gameId, jobType, profile) {
  const subDims = SUB_DIMENSIONS[gameId];      // 该游戏的子维度列表
  const jobIdeal = IDEAL_PROFILES[jobType];     // 该岗位的理想剖面
  const weights  = SUB_WEIGHTS[gameId][jobType]; // 子维度权重
  
  let totalWeight = 0, weightedSum = 0;
  for (const dim of subDims) {
    const score = profile[dim];
    const direction = jobIdeal[dim].direction;
    const weight = weights[dim];
    weightedSum += mapSubDimension(score, direction, weight);
    totalWeight += weight;
  }
  return weightedSum / totalWeight; // 0-1
}
```

#### 步骤 3：最终线性得分

```javascript
function linearFinalScore(jobType, userProfile) {
  const gameWeights = GAME_WEIGHTS[jobType];
  return (
    gameLevelScore("G1", jobType, userProfile) * gameWeights.G1 +
    gameLevelScore("G2", jobType, userProfile) * gameWeights.G2 +
    gameLevelScore("G3", jobType, userProfile) * gameWeights.G3 +
    gameLevelScore("G4", jobType, userProfile) * gameWeights.G4
  ) * 100; // 0-100
}
```

### 5.3 轨道二：TOPSIS 距离法

#### 核心思想

TOPSIS 是 Technique for Order Preference by Similarity to Ideal Solution 的缩写。核心逻辑：

> 计算用户到"最佳剖面（正理想解）"的距离 d⁺，和到"最差剖面（负理想解）"的距离 d⁻。贴近度 C = d⁻/(d⁺+d⁻)，C 越大越接近理想。

#### 算法步骤

```
Step 1: 构建决策矩阵 D[m×n]
        m = 4 个岗位，n = 14 个维度
        每个元素 D[i][j] = 用户在第j维对岗位i的匹配度（来自步骤5.2的mapSubDimension输出）

Step 2: 归一化
        R[i][j] = D[i][j] / sqrt(Σ D[k][j]²)   （向量归一化）

Step 3: 加权
        V[i][j] = R[i][j] × w[j]                （w[j] 为维度j的全局权重）

Step 4: 确定理想解
        A⁺ = { max(V[i][j]) for each j }        （正理想解）
        A⁻ = { min(V[i][j]) for each j }        （负理想解）

Step 5: 计算欧氏距离
        d⁺[i] = sqrt( Σ (V[i][j] - A⁺[j])² )
        d⁻[i] = sqrt( Σ (V[i][j] - A⁻[j])² )

Step 6: 计算贴近度
        C[i] = d⁻[i] / (d⁺[i] + d⁻[i])        （0-1，越大越匹配）

Step 7: 排序
        按 C[i] 降序排列
```

#### JS 实现参考

```javascript
function topsis(userProfile, jobTypes) {
  const n = 14; // 维度数
  const m = jobTypes.length;
  
  // Step 1: 构建决策矩阵
  const D = jobTypes.map(jt => 
    getMatchVector(userProfile, jt) // 14维匹配度向量
  );
  
  // Step 2: 向量归一化
  const colSums = Array(n).fill(0);
  for (let j = 0; j < n; j++)
    for (let i = 0; i < m; i++)
      colSums[j] += D[i][j] ** 2;
  
  const R = D.map(row => 
    row.map((val, j) => val / Math.sqrt(colSums[j]))
  );
  
  // Step 3: 加权（使用全局维度权重）
  const V = R.map(row => 
    row.map((val, j) => val * DIMENSION_WEIGHTS[j])
  );
  
  // Step 4: 理想解
  const Aplus = Array(n).fill(0).map((_, j) => 
    Math.max(...V.map(row => row[j]))
  );
  const Aminus = Array(n).fill(0).map((_, j) => 
    Math.min(...V.map(row => row[j]))
  );
  
  // Step 5-6: 距离 + 贴近度
  return V.map((row, i) => {
    const dPlus  = Math.sqrt(row.reduce((s, v, j) => s + (v - Aplus[j]) ** 2, 0));
    const dMinus = Math.sqrt(row.reduce((s, v, j) => s + (v - Aminus[j]) ** 2, 0));
    return {
      jobType: jobTypes[i],
      closeness: dMinus / (dPlus + dMinus),
      dPlus, dMinus
    };
  }).sort((a, b) => b.closeness - a.closeness);
}
```

### 5.4 轨道三：AI 辅助评估（加分项）

```javascript
async function aiAssistedScore(userProfile, topsisResults) {
  // 仅在以下情况触发 AI：
  // 1. 线性排名与 TOPSIS 排名不一致（差异≥2位）
  // 2. 第一名贴近度 < 0.6（不存在明显最优）
  
  const needAI = checkDiscrepancy(topsisResults, linearResults);
  if (!needAI) return null;
  
  const prompt = buildAIPrompt(userProfile, topsisResults);
  const aiResponse = await callLLM(prompt);
  
  return {
    confidence: aiResponse.confidence,   // AI 对每个岗位的置信度
    reasoning: aiResponse.reasoning,     // 推荐理由
    flags: aiResponse.flags              // 需要注意的问题
  };
}
```
